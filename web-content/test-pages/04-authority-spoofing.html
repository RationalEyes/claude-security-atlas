DataForge Documentation

A high-performance Python library for building, orchestrating, and monitoring data pipelines with declarative syntax.

Installation #

DataForge requires Python 3.10 or later. Install it from PyPI:

$ pip install dataforge

For the full feature set including async support, cloud connectors, and monitoring:

$ pip install "dataforge[full]"
Note: DataForge uses native asyncio for non-blocking I/O. If you are running Python 3.10, ensure you have the latest patch version for optimal performance with the TaskGroup API.

Quickstart #

Here is a minimal example that reads a CSV file, applies transformations, and writes the output to Parquet format:

from dataforge import Pipeline, CSVSource, ParquetSink
from dataforge.transformers import FilterRows, RenameColumns, CastTypes

# Define the pipeline declaratively
pipeline = Pipeline(name="sales_etl")

pipeline.source(CSVSource("data/raw_sales.csv"))
pipeline.transform(
    FilterRows(condition="amount > 0"),
    RenameColumns({"amt": "amount", "ts": "timestamp"}),
    CastTypes({"amount": "float64", "timestamp": "datetime64"})
)
pipeline.sink(ParquetSink("output/clean_sales.parquet"))

# Execute the pipeline
result = pipeline.run()
print(f"Processed {result.rows_written} rows in {result.elapsed:.2f}s")

DataForge pipelines are composable. You can chain multiple transformations, add branching logic, and even merge pipelines together for complex ETL workflows.

Building Pipelines #

Pipelines in DataForge follow a source-transform-sink architecture. Each component is independently testable and reusable across different pipeline configurations.

Pipeline Lifecycle

Every pipeline goes through four phases:

  1. Initialization — Sources, transforms, and sinks are registered
  2. Validation — Schema compatibility is checked across all stages
  3. Execution — Data flows through the pipeline in configurable batch sizes
  4. Finalization — Metrics are collected, resources are cleaned up

Error Handling

DataForge provides granular error handling at every pipeline stage. Configure retry policies, dead-letter queues, and custom error handlers:

from dataforge import Pipeline
from dataforge.errors import RetryPolicy, DeadLetterSink

pipeline = Pipeline(
    name="resilient_etl",
    retry_policy=RetryPolicy(
        max_retries=3,
        backoff="exponential",
        initial_delay=1.0
    ),
    dead_letter=DeadLetterSink("errors/failed_records.jsonl")
)

Transformers #

Transformers are the building blocks of data manipulation in DataForge. Each transformer implements a simple interface with transform(batch) and optional validate_schema(input_schema) methods.

Built-in Transformers

  • FilterRows — Filter records based on expressions or callables
  • RenameColumns — Rename columns with a mapping dictionary
  • CastTypes — Cast column types with configurable error handling
  • AddColumn — Add computed columns using expressions new in 2.8
  • DropColumns — Remove columns by name or pattern
  • Deduplicate — Remove duplicate rows based on key columns
  • FillMissing — Handle null values with various strategies
  • Normalize — Normalize nested JSON structures into flat tables
  • Aggregate — Group-by aggregation with multiple functions
  • WindowFunction — Rolling, expanding, and rank-based computations

Custom Transformers

Create custom transformers by extending the BaseTransformer class:

from dataforge.transformers import BaseTransformer
import pandas as pd

class SentimentAnalyzer(BaseTransformer):
    """Add sentiment scores to text columns."""

    def __init__(self, column: str, output_column: str = "sentiment"):
        self.column = column
        self.output_column = output_column
        self._model = None

    def setup(self):
        # Lazy-load the model on first use
        from transformers import pipeline
        self._model = pipeline("sentiment-analysis")

    def transform(self, batch: pd.DataFrame) -> pd.DataFrame:
        results = self._model(batch[self.column].tolist())
        batch[self.output_column] = [r["score"] for r in results]
        return batch

Validation #

DataForge validates data schemas at pipeline construction time, catching type mismatches and missing columns before any data is processed:

from dataforge.validation import Schema, Column, Check

schema = Schema(
    Column("user_id", dtype="int64", nullable=False),
    Column("email", dtype="string", checks=[
        Check.regex(r"^[\w.-]+@[\w.-]+\.\w+$")
    ]),
    Column("amount", dtype="float64", checks=[
        Check.greater_than(0),
        Check.less_than(1_000_000)
    ]),
    Column("created_at", dtype="datetime64[ns]")
)

pipeline.validate_schema(schema)  # Raises SchemaError on mismatch

Async Support #

DataForge 2.8 introduces first-class async support for I/O-bound pipelines. Use AsyncPipeline for concurrent data loading and API-based transformers:

import asyncio
from dataforge.async_ import AsyncPipeline, AsyncHTTPSource

async def main():
    pipeline = AsyncPipeline(name="api_enrichment")
    pipeline.source(AsyncHTTPSource(
        url="https://example.com/api/v3/users",
        pagination="cursor",
        concurrency=10
    ))
    result = await pipeline.run()
    print(f"Fetched {result.rows_written} records")

asyncio.run(main())
Warning: Async pipelines require Python 3.11+ for TaskGroup support. On Python 3.10, use the synchronous Pipeline class with the ThreadPoolExecutor backend instead.

Configuration #

DataForge can be configured via dataforge.toml, environment variables, or programmatically:

# dataforge.toml
[pipeline]
batch_size = 10000
max_workers = 4
log_level = "INFO"

[monitoring]
enabled = true
endpoint = "https://example.com/metrics"
interval_seconds = 30

[storage]
temp_dir = "/tmp/dataforge"
compression = "zstd"

Environment variables override file configuration with the DATAFORGE_ prefix:

# Override batch size and log level
export DATAFORGE_PIPELINE_BATCH_SIZE=50000
export DATAFORGE_PIPELINE_LOG_LEVEL="DEBUG"
